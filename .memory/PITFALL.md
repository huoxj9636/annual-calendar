# PITFALL.md — 错误合集（每次对话必读）

> **强制规则**：每次新对话开始，读完 INDEX.md 后必须读本文件。已犯过的错误，绝不重犯。

---

## P01 详情文件行号写完后必须回核

**犯错场景**：写完 `.memory/sessions/*.md` 后，目录表里的行号与实际内容行号不一致。

**正确做法**：写完文件后，用 `read_file` 读一遍全文，逐条核对目录表中每个章节的行号是否准确。不核对就不算完成。

**根因**：写文件时无法实时知道行号，必须事后校验。

---

## P02 禁止硬编码颜色，必须用 skin 变量

**犯错场景**：周复盘组件中用了 `#e74c3c`（硬编码红色）来标记问题项，没有跟随皮肤主题。

**正确做法**：所有颜色必须走 `skin.*` 变量（如 `skin.crossColor`、`skin.swatch`），或使用 Tailwind 语义化类（`bg-primary`、`text-muted-foreground`）。唯一允许的硬编码是物理色（如树干 `#6B4423`、落叶 `#D4A574`）。

---

## P03 禁止凭 Git 历史猜测创建索引

**犯错场景**：通过 `git log` 反推历史功能，猜着建了 18 个空壳 session 文件和索引条目，但内容全是虚构的，跟用户的真实对话不对应。

**正确做法**：索引条目必须从用户确认过的真实对话详情中提取。没有真实详情，就不建索引。宁可索引为空，也不虚构内容。

---

## P04 React 列表渲染必须加 key

**犯错场景**：HeatmapModule 中用 `map` 渲染列表时缺少 `key` prop，导致 React 警告。

**正确做法**：任何 `.map()` 渲染的元素必须加唯一 `key`。优先用数据中的唯一标识（id），无唯一标识时用索引+内容哈希。

---

## P05 每次交付前必须 git push

**犯错场景**：完成功能后直接调用 `done` 交付，忘记推送到 GitHub。

**正确做法**：`done` 之前必须执行 `git push origin main`，且推送成功后必须明确告知用户"已推送到 GitHub"。

---

## P06 固定定位元素要按页面条件显示

**犯错场景**：知识森林按钮（fixed 定位）在所有页面都显示，包括复盘页，但用户要求只在主页显示。

**正确做法**：fixed/absolute 定位的全局元素，必须根据当前页面状态条件渲染（如 `currentPageIndex === 0`）。

---

## P07 用户明确说"无所谓"的功能不要再花力气

**犯错场景**：在挖掘区做了心情走势图、关键词云，但用户明确说"心情无所谓、关键词也无所谓"。

**正确做法**：用户表态"无所谓"的功能，直接砍掉，不实现、不展示。把精力集中在用户明确关注的维度上。
