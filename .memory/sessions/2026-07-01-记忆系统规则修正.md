## Q1/A1 session文件应每个对话独立

**用户问**：我又发现一个问题。你说的那个就是我看一下啊。就是 session memory 里面的 session 你那个周复盘时间轴。对吧？你是你是一个文件。其实那个是基于我们当前对话所创建的一个文件。我最开始的意思就是每一个对话都要有一个独立的文件，只是说它这个文件里面的内容是当前会话所有的内容嘛。那因为我们刚刚提到，就是说 是不是就在已创建的这个文件里面添加内容？那很明显这个详情文件肯定不是的，详情文件是要每个对话独立创建出来一个的。对吧？你这个要搞清楚的。你看你是不是要在规则里面什么都改一下，然后同时让他们把这个事情搞清楚，不要弄混。

**AI答**：用户指出一个根本性区分错误——session详情文件应该是每个对话独立一个文件，绝不能把新对话的内容追加到旧对话的文件里。之前的规则"同一主题延续对话追加到已有文件"是错的。正确规则是：
- INDEX.md / PITFALL.md → 跨对话追加（所有对话共享）
- sessions/*.md → 每个对话独立一个文件（绝不追加到别人的文件里）

需要修正 AGENTS.md 中的规则、同步更新技能包模板、把错误追加的内容从旧文件迁移到新文件、追加PITFALL条目。
