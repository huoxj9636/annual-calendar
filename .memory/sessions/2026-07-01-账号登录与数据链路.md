## Q12/A12（7/1）登录弹窗自动弹出修复（apiFetch GET静默）

**用户原话**：
> 那我之前说只有在输入的时候、保存的时候才会提示登录，那为什么现在这个登录框老是自动跳出来？

**AI 回答**：
根因：apiFetch 在任何请求（包括页面加载时的 GET 请求）session 不存在时都触发 requireLogin()。
修复：修改 api-client.ts，只有 POST/PUT/DELETE（写入操作）才触发 requireLogin()，GET（读取操作）静默返回 null。

## Q11/A11（7/1）全面数据链路检查

**用户原话**：
> 就是说增加了这个账号功能之后，很多内部的这些链路数据是不通的...

**AI 回答**：
逐一检查所有数据保存到数据库的链路：

1. **calendar_overrides**：前端 toggleCell 只更新本地状态，没有调用 API 保存 → 添加 API 保存调用
2. **calendar_notes/drawings**：API 添加 id 生成（nanoid）
3. **knowledge_trees**：只有 localStorage 无 API → 记录未修复，后续需创建 API 和数据库表
4. 修复 TypeScript 类型错误（toggleCell newValue 类型不匹配，显式声明 `newValue as DayOverride`）

## Q9/A9（7/1）saveReview函数bug

**用户原话**：
（Console Error 显示 [saveReview] Failed: 'null response'）

**AI 回答**：
根因：`res.catch()` 错误，Response 对象没有 catch 方法。apiFetch 返回的是 Response 而非 Promise，错误处理逻辑不匹配。
修复：正确处理 apiFetch 返回值，用 try-catch 包裹而非 .catch()。

## Q8/A8（7/1）daily_review保存失败(upsert→delete+insert)

**用户原话**：
> 我现在操作下来，就是在我每日复盘里面，今日完成和明日待办里面填上内容之后呢。它不仅那个勾没有打对号，甚至我重新打开的时候我的数据都没有保存。

**AI 回答**：
根因：upsert 需要 (user_id, year, month, day) 唯一约束，但数据库没有这个约束，导致 upsert 实际上每次都 insert 新行而非更新。
修复：将 upsert 改为 delete + insert 模式。同样修复了 import-review API。

## Q5/A5（7/1）部署NOT NULL约束错误

**用户原话**：
（多次部署错误日志）

**AI 回答**：
根因：user_id 列有 null 值但 schema 要求 NOT NULL。
修复：
1. 生产环境 `UPDATE ... SET user_id = 'legacy' WHERE user_id IS NULL`
2. schema 改为 nullable（去掉 .notNull()）
3. 手动给生产表添加 user_id 列（exec_sql product 只支持 SELECT，需要其他方式）

注意：修改 schema 后需要重新部署才能生效。

## Q2/A2（7/1）MigrationToast每次刷新弹出

**用户原话**：
> 那个，点开知识森林之后，它每次刷新它都会提醒有数据要合并。可是这个问题我反复跟你沟通了很多次...

**AI 回答**：
根因：localStorage key 不一致（读写后缀不匹配）。
- 读用的是带 userId 后缀的 key
- 写用的是全局 key（不带后缀）
- 导致每次读到的 dismissed 状态都是 undefined，toast 重新弹出

修复：
1. 统一使用全局 key（不带 userId 后缀）
2. dismissed 状态从 useRef 改为 useState（确保重渲染）
3. 修改文件：sync-provider.tsx

## Q1/A1（7/1）登录按钮位置与可见性

**用户原话**：
> 那个右上角的登录按钮，你虽然给它放到头图一起，但是你也放到最顶端，让它显示可见啊，现在看不到。

问题2：按钮位置需要多次调整。
修复：absolute 定位，向下移动 25+8+10=43px。

修改文件：
