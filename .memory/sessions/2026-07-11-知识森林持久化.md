## Q1/A1 定位圆环对齐问题(1)

**用户**：我的树里面统计的是所有页面的数的总量。所以当我点击这一个定位的时候，我的意思就是说 如果 我当前在第一页。而我是 选择的那个树在第二页，那我定位其实它也会要跳到第二页去锁定我选中的那个树。

**AI**：理解，点击定位时不仅要跳转到对应页码，还要用焦点圆环锁定那棵树。实现：在 `handleFocusTree` 中先计算目标树在哪个分页，然后 setForestPage 跳转，同时触发焦点圆环的入场动画，让用户看到树被高亮锁定。

**关键修改**：
- `focusTreeId` 状态驱动焦点圆环
- `handleFocusTree` 自动计算分页索引并跳转
- 圆环动画：`animate-scale-in` CSS 类 + 200ms 延迟后显示

## Q4/A4 定位圆环对齐问题(4)

**关键修改**：

- 所有圆环的 `left`/`bottom` 值从 `displayX`/`displayY` 直接计算

## Q9/A9 树位置集中问题

**用户**：你那个知识森林里面，我新创建的这个树啊，为什么我重新打开这个知识森林这个页面之后呢，它所有的这个树全部都集中在一起。应该是我创建最后拖拽到哪个位置，我打开知识森林这个页面加载出来，它还应该在哪个位置。

**AI**：问题出在 POST 路由中 `position?.x ?? 50` 导致所有没有指定位置的新树都默认获得 `position: {x:50, y:50}`，全部叠在中心。修复：POST 改为 `position?.x ?? null`，GET 返回时跳过 position_x/y 为 null 的字段。同时 scale 字段 DB 存 100 但前端用 1.0，GET 返回时除以 100 做转换。

**关键修改**：
- POST 路由 `position?.x ?? 50` → `position?.x ?? null`
- GET 路由只返回 `position` 字段当 position_x 不为 null 时
- GET 返回 `scale` 除以 100
- PUT 也可更新位置/名称/链接/节点

**关键修改**：

## Q11/A11 树固定左上角

**用户**：在开发环境里面，我把树的位置调整之后，我关闭页面重新进入，它还是居中的。并不是我最终拖拽的位置。你什么意思？就是树会固定到你那个 left 15%、 bottom 72%这个位置上，是吗？我的意思是我最后拖到哪，它就拖，它就固定在哪。我现在我测试下来，它跑到左上角去了。每次刷新无论我拖到哪，它都固定在左上角去了，它又居中。变为固定到左上角去了。不应该是这样的。

**AI**：追查发现 PUT /api/trees/[id] 返回 500。app.log 显示：
```
PUT /api/trees/ade82485-22e3-410a-b177-4225e26dd94c 500 in 65ms
```
终端 curl 测试成功（200），但浏览器调用失败（500）。原因是 DB 表 `knowledge_trees` 启用了 RLS 但没有公开策略，`getSupabaseClient()` 如果使用 anon key 会被 RLS 阻断。修复：
1. PUT 路由添加 try/catch 异常捕获
2. 添加公开 RLS 策略 `allow_public_all`（FOR ALL TO anon, USING true, WITH CHECK true）
3. 验证：test_run 3 个接口全部通过（GET 200 + POST 201 + PUT 200）

**关键修改**：
- PUT route 添加 try/catch 和详细错误日志
- 添加公开 RLS 策略

## Q12/A12 用户报告仍500，修复Supabase单例

**用户**：Console Error: 保存位置失败: 500

**AI**：检查 app.log 发现 PUT 500 是间歇性出现的——有时 200，有时 500。根因是 `loadEnv()` 中的 `execSync('python3 ...')` 在 Next.js 运行时偶尔失败，导致 Supabase 客户端初始化异常。修复：
1. Supabase 客户端改为单例模式（`getSupabaseClient()` 缓存 client）
2. `loadEnv()` 增加容错，Python 执行失败时标记 `envLoaded = true` 避免反复重试
3. 缓存 `getSupabaseCredentials()` 的结果

**关键修改**：
- `src/storage/database/supabase-client.ts` — 单例 client + loadEnv 容错

## Q13/A13 用户报告仍500，localStorage兜底

**AI**：仍然 500，说明 Supabase 间歇性失败无法完全消除。改为"前端兜底"策略：

## Q15/A15 localStorage兜底+重试
