## Q1/A1 — 书房功能需求沟通与实现 (7/21)

**用户**：想在知识森林模块再增加一个 tab 页，主要是记录书单，可以给微信读书链接跳转到微信读书去。

**AI 构思方案**：
- Tab 命名"书房"，与"我的森林"并列
- 顶部统计条（总计/在读/想读/已读）
- 书单标签栏（横向滚动，可新建/编辑/删除书单）
- 图书按状态分组（在读→想读→已读），卡片网格展示
- 每张卡片：封面图、书名、作者、状态标签、悬停显示"去阅读"按钮
- FAB 添加新书，弹窗表单支持粘贴微信读书链接自动解析

**用户确认**：
1. 封面图：不接受文字色块，优先自动解析微信读书封面，失败则手动填 URL
2. 阅读状态：想读/在读/已读，三种就够
3. 数据存储：Supabase 数据库
4. 链接解析：后端自动解析书名、作者、封面

**用户问 Supabase 是什么**：解释为项目已在用的云端数据库，知识树也存在里面。

**实现内容**：
- 数据库：`book_lists` 表 + `books` 表
- API：`/api/book-lists`（CRUD）、`/api/books`（CRUD）、`/api/parse-weread`（解析微信读书链接）
- 前端：`reading-room.tsx` 组件 + `knowledge-panel.tsx` 加 Tab 切换
- 所有测试通过（lint、ts-check、API 冒烟测试）
