# DESIGN.md

## 知识森林（Knowledge Forest）

### 气质与意象

**意象锚点**：清晨薄雾笼罩的山谷森林，第一缕阳光从树冠斜洒下来，远山如黛，露珠在草尖闪烁，偶有一片金黄的叶子旋转落下。整个森林不是卡通的"奇幻森林"，而是有时间感的、沉静的、让人想深吸一口的森林。

**反例自检**（要绕开的方案）：
- ❌ 大饱和度纯绿 + 卡通 emoji 大树
- ❌ 蓝紫渐变（典型 AI 味）
- ❌ 一片绿油油平面（缺层次）
- ❌ 玻璃拟态卡片堆叠（不契合自然主题）

### 视觉策略

**核心场景：森林全景（Forest Panorama）**
- 横向展开的山谷全景，像在山脊上俯瞰一片林地
- **四层透视**：晨空（淡暖白）→ 远山（深绿剪影）→ 中景山丘（橄榄/苔藓）→ 前景草地（嫩绿）
- 树木按"知识节点"数量成长为不同阶段：幼苗 → 小树 → 成树 → 参天
- 5 种知识类型（根/干/枝/叶/果）映射为不同树种：橡树/杉木/枫/嫩枝/樱花
- 飘落的叶子（下落 + 旋转 + 透明度淡出）
- 缓慢横向移动的云朵
- 阳光透过树冠的光斑（半透明白色圆斑）

**单棵树特写（Tree Closeup）**
- 大图：单棵树的剪影 + 知识类型徽章浮在树枝上
- 阶段进度条（幼苗→小树→成树→参天）
- 每种类型的知识以"果实/树叶"形式聚类显示

### 配色方案

**跟随皮肤主题色**（用户可在 8 套皮肤中选择）：
- 主强调色 = `skin.swatch`（用户当前选择）
- 面板背景 = `skin.panelBg`（带极淡森林绿叠加）

**森林场景专用色**（半透明叠加在主题色上）：
- 晨空：`from-muted/30 to-background`（淡暖白渐变）
- 远山剪影：`skin.swatch` + 透明度 0.12-0.18
- 中景山丘：`skin.swatch` + 透明度 0.25-0.35
- 草地：`skin.swatch` + 透明度 0.15-0.40（前景到后景递增）
- 树干：暖棕色 `#6B4423`（物理色，不变）
- 飘落叶：暖黄 `#D4A574`（物理色）
- 光斑：`background` + 透明度 0.40

**严格遵循 shadcn 规范**：
- 所有 UI 元素（按钮/输入框/标签/进度条）用 `bg-primary`、`text-primary-foreground`、`bg-card` 等语义化变量
- 森林场景作为"插画层"独立于 UI 系统，使用物理色 + 透明度叠加
- 不用 Hex 硬编码（除"树干/落叶"等物理色）

### 字体排版

- 标题：Noto Serif SC（衬线，与自然主题契合，已有）
- 正文：PingFang SC（项目默认）
- 关键数据（能量、阶段）：font-mono（等宽，突出数字）
- 阶段标签：uppercase + tracking-widest（小字距宽，工艺感）

### 动效与交互

**主运动性格**：沉静的、有呼吸感的自然律动
- 树摇曳：3 度角，3-5s 周期，正弦曲线，staggered 错开（避免整齐划一）
- 云朵漂浮：30s 横向漂移，线性
- 飘落叶：8-12s 一次，随机起点 + 随机终点（横向 0-100%），旋转 360 度，透明度淡出
- 光斑浮动：subtle 透明度脉动 4s
- 进度条增长：500ms ease-out
- 树木点击：scale 1 → 1.05 → 1（200ms spring）

**鼠标交互**：
- 悬停树木：摇曳幅度 +50%，光斑变亮
- 悬停节点卡：上移 2px，阴影加深
- 添加节点：能量球从 FAB 飞向对应树（粒子效果）

### 页面结构

**Tab 1 - 我的森林（My Forest）**：
1. 顶部统计条（4 个数据块）：树总数 / 总能量 / 最高阶段 / 本月新增
2. 森林全景画布（占主要空间）：
   - 远山 → 中景 → 草地 → 树木
   - 树木按节点数量分布
   - 飘落叶/云朵/光斑
3. 我的树列表（紧凑横排）：阶段图标 + 树名 + 节点数 + 进度
4. FAB：「种一棵新树」

**Tab 2 - 好友森林（Friends Forest）**：
1. 同样的森林全景布局
2. 好友头像作为"不同种类的树"分布在森林中
3. 点击进入好友主页（新窗口）

**单棵树特写（点击树后进入）**：
1. 顶部：返回 + 树名 + 行业/描述
2. 大图区：单棵树的剪影（占 40% 高度）+ 5 种类型的徽章浮在对应位置
3. 阶段进度条（幼苗 → 小树 → 成树 → 参天）
4. 知识分组（5 个卡片，按类型）：每张卡片显示该类型的知识列表

### 设计禁忌（明确的"不要做"）

1. ❌ 不要用 emoji 🌳🌲🌴 作为树木图标（用 lucide-react 的 TreePine、TreeDeciduous、Sprout、Sapling、Cherry、TreePalm）
2. ❌ 不要蓝紫/紫色系渐变（AI 味）
3. ❌ 不要纯平面单一绿色（要 4 层透视）
4. ❌ 不要用 `[#xxxxxx]` 硬编码（除物理色如树皮、落叶）
5. ❌ 不要 `[8px]` 这种硬编码圆角（用 rounded-md/lg）
6. ❌ 不要静态展示（必须动效：摇曳/飘落/光斑）
7. ❌ 不要破坏现有数据结构（localStorage keys 保持不变：knowledge-trees、knowledge-bookmarks）
8. ❌ 不要丢失 5 种知识类型（root/trunk/branch/leaf/fruit 保持）
9. ❌ 不要过密的卡片堆叠
10. ❌ 不要"卡片森林"风格（应有真正"森林"感，不是 UI 元素的堆叠）

### 与蚂蚁森林的差异化

- 蚂蚁森林是公益种树（用户攒能量→种真树），我们是知识管理
- **保留蚂蚁森林的"灵魂"**：全景视野 / 树木成长 / 能量体系 / 好友互动
- **去蚂蚁森林的"商业"元素**：没有环保证书、没有公益组织
- **加入知识管理特色**：5 种知识类型映射树种、单棵树知识图鉴
